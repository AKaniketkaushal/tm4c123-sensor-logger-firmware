# TM4C123 Sensor Logger Firmware

FreeRTOS-based firmware for the TM4C123GXL that reads multiple sensors, sends data over BLE, exposes a UART debug console, logs records to internal EEPROM, and supervises task health with a watchdog.

## Features

- BMP180 temperature and pressure acquisition
- MPU6050 accelerometer and gyroscope acquisition
- QMC5883L/HMC5883L-compatible magnetometer acquisition on the I2C bus used by this project
- BLE transmit/receive task split using FreeRTOS queues and stream buffers
- UART debug console with runtime commands
- Internal EEPROM record storage
- Software watchdog task layered over the TM4C123 hardware watchdog
- I2C bus recovery and basic communication fault reporting

## Architecture Summary

The firmware creates separate FreeRTOS tasks for BLE send, BLE receive, debug console, debug actions, sensor handling, and watchdog supervision.

Sensor commands enter through BLE or UART, are dispatched through queues, and are handled by `SENSOR_TASK` or `DEBUG_TASK` depending on the source and purpose. Sensor samples can be transmitted live, printed on the debug UART, and stored as records in internal EEPROM.

The watchdog manager expects each major task to report liveness within the configured time window before it refreshes the hardware watchdog.

## Runtime Notes

- Internal EEPROM is initialized on boot
- I2C2 is used for the sensors
- UART0 is used for debug output / debug command input
- UART1 is used for the BLE module

## What This Project Demonstrates

- RTOS task design on a Cortex-M MCU
- ISR-to-task communication using stream buffers and queues
- Multi-sensor I2C integration
- Embedded watchdog supervision
- EEPROM-backed logging
- Debugging and failure-handling on real hardware

## Current Limitations

- Some fault recovery paths are still being improved
- Error/status metadata in EEPROM records can be expanded further
- Logging and debug output are still relatively synchronous
- Sensor recovery and calibration protection can be hardened more

## Roadmap Ideas

- Add explicit error/status fields to EEPROM records
- Move logging to a non-blocking queued logger task
- Improve sensor re-initialization policy after repeated failures
- Add a more structured BLE command/response protocol
- Persist calibration/configuration data in EEPROM

## Showcase Notes

This is a personal embedded firmware project focused on real hardware bring-up, task coordination, sensor integration, and robustness improvements. It is intended to show practical embedded systems work rather than polished production firmware.
